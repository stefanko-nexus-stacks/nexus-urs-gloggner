## CloudBeaver

![CloudBeaver](https://img.shields.io/badge/CloudBeaver-3776AB?logo=dbeaver&logoColor=white)

**Web-based database management tool**

CloudBeaver is an open-source cloud database management tool built on DBeaver. It provides a web-based interface for managing databases without installing any software. Features include:
- Support for 30+ databases (PostgreSQL, MySQL, SQL Server, Oracle, etc.)
- SQL editor with syntax highlighting and auto-completion
- Visual query builder for non-SQL users
- Data export in multiple formats (CSV, JSON, XML)
- Role-based access control
- Connection management and sharing

| Setting | Value |
|---------|-------|
| Default Port | `8978` |
| Suggested Subdomain | `cloudbeaver` |
| Public Access | No (database credentials) |
| Website | [dbeaver.com/cloudbeaver](https://dbeaver.com/cloudbeaver/) |
| Source | [GitHub](https://github.com/dbeaver/cloudbeaver) |

### Configuration

- **Admin user:** `nexus-cloudbeaver`
- **Admin password:** Auto-generated by OpenTofu, stored in Infisical as `CLOUDBEAVER_PASSWORD`

### Usage

#### First-time login

Navigate to `https://cloudbeaver.<your-domain>`. Log in with:

- **Username:** `nexus-cloudbeaver`
- **Password:** from Infisical → folder `cloudbeaver` → key `CLOUDBEAVER_PASSWORD`

#### Add the Nexus Postgres connection

Connections can only be created when logged in as an admin. Once added, they are available to all users.

1. After logging in as admin, click the **+** button in the top-left panel
2. Select **New Connection**
3. Select **PostgreSQL** as the driver
4. Fill in the connection details:

| Field | Value |
|---|---|
| Host | `postgres` |
| Port | `5432` |
| Database | `postgres` |
| Username | `nexus-postgres` |
| Password | from Infisical → folder `postgres` → key `POSTGRES_PASSWORD` |
| Connection name | `Nexus Postgres` (or any name) |

5. Click **Test** to verify, then **Create**
6. Under **Access**, grant access to any users or teams who should see this connection

#### Run SQL queries

Open the **SQL Editor** from the top menu. Before running a query, select the active connection from the **"Select connection"** dropdown at the top — the editor will not execute queries until a connection is set as active.

Then run SQL against any schema your pipeline has created, for example:

```sql
SELECT article, timestamp, views
FROM wikipedia.pageviews
ORDER BY article, timestamp
LIMIT 20;
```
