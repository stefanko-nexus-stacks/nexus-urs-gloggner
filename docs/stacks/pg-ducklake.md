## pg_ducklake

![pg_ducklake](https://img.shields.io/badge/pg__ducklake-336791?logo=postgresql&logoColor=white)

**PostgreSQL with the DuckLake extension - SQL-native lakehouse with S3 storage.**

pg_ducklake is PostgreSQL 18 with the [DuckLake](https://ducklake.select) extension preinstalled. It provides a SQL-native lakehouse format where the catalog and metadata live in PostgreSQL, while table data is stored as Parquet files in S3-compatible object storage. Standard PostgreSQL tools (pgAdmin, CloudBeaver, Superset, Marimo, Adminer) connect to it normally and can query DuckLake tables alongside regular Postgres tables.

| Setting | Value |
|---------|-------|
| Default Port | `5432` |
| Internal Only | Yes (no web UI, no Cloudflare Tunnel) |
| Protocol | PostgreSQL wire protocol |
| Default User | `nexus-pgducklake` |
| Default Database | `ducklake` |
| Storage | Hetzner Object Storage (S3) when configured, local volume otherwise |
| Website | [pgducklake.select](https://pgducklake.select) |
| Source | [GitHub](https://github.com/relytcloud/pg_ducklake) |

> ⚠️ **Not yet production-ready.** Upstream marks pg_ducklake as "under high development." Use for self-hosted lab and experimentation, not for critical workloads.

### Credentials

All credentials are available in Infisical under the `pg-ducklake` folder:

- `PG_DUCKLAKE_USERNAME` → `nexus-pgducklake`
- `PG_DUCKLAKE_PASSWORD` → generated by OpenTofu
- `PG_DUCKLAKE_DATABASE` → `ducklake`
- `PG_DUCKLAKE_S3_BUCKET` → `nexus-<domain>-pgducklake` (when Hetzner Object Storage is configured)

### Connection String

From any container on the `app-network`:

```
postgresql://nexus-pgducklake:<password>@pg-ducklake:5432/ducklake
```

For external access, use an SSH tunnel:

```bash
ssh nexus -L 5433:pg-ducklake:5432
```

Then connect locally to `postgresql://nexus-pgducklake:<password>@localhost:5433/ducklake`.

### Sample Workflow

Connect via pgAdmin (or any PostgreSQL client) and create a DuckLake table:

```sql
-- Verify the extension is loaded
SELECT * FROM pg_extension WHERE extname LIKE '%duck%';

-- Check current default storage path (configured by deploy.sh)
SHOW ducklake.default_table_path;

-- Create a DuckLake table - data is automatically stored as Parquet in S3
CREATE TABLE events (
    id BIGINT,
    event_type TEXT,
    occurred_at TIMESTAMPTZ
) USING ducklake;

INSERT INTO events VALUES
    (1, 'login',  NOW()),
    (2, 'logout', NOW());

-- Query the data
SELECT event_type, COUNT(*) FROM events GROUP BY event_type;
```

### Storage Configuration

When the spin-up workflow runs, deploy.sh generates `/docker-entrypoint-initdb.d/00-ducklake-bootstrap.sql` and applies it on first init. After every spin-up the same script is also re-applied via `docker exec` to handle credential rotation on existing data volumes.

**With Hetzner Object Storage** (default when configured):
- Bucket: `nexus-<domain>-pgducklake` (created by OpenTofu, persists through teardown)
- S3 secret created via `duckdb.create_simple_secret(...)`
- `ducklake.default_table_path` set to `s3://<bucket>/`

**Without Hetzner Object Storage** (local fallback):
- Parquet files stored in the `pgducklake-lake` Docker volume at `/var/lib/ducklake/`
- Useful for testing but defeats the lakehouse purpose

### Caveats

- **Not production-ready** per upstream maintainers
- **Local-to-S3 migration**: If you first deploy without Hetzner Object Storage and later add it, existing local Parquet files will be orphaned. Plan storage choice up front
- **Credential rotation**: Rotating Hetzner S3 keys requires re-running spin-up so the persistent secret in Postgres is refreshed
- **Image tag**: `pgducklake/pgducklake:18-main` is a rolling development tag, not a semver release

### Cross-Tool Access

Since pg_ducklake speaks the standard PostgreSQL wire protocol, you can query DuckLake tables from any tool already in Nexus-Stack:

| Tool | Use Case |
|------|----------|
| **pgAdmin** | Browse catalog, run ad-hoc SQL, inspect query plans |
| **CloudBeaver** | Multi-database explorer with the same DuckLake connection |
| **Superset** | Build dashboards on DuckLake tables (add as PostgreSQL connection) |
| **Marimo** | Interactive Python notebooks against DuckLake via psycopg2 |
| **Trino / Spark / Flink** | Federated queries combining DuckLake with other sources |
