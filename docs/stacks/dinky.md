## Dinky

![Dinky](https://img.shields.io/badge/Dinky-1677FF?logo=apacheflink&logoColor=white)

**Web-based Flink SQL IDE with auto-completion, job management, and real-time debugging**

Dinky is a real-time data development platform based on Apache Flink. It provides a browser-based SQL editor for writing and executing Flink SQL queries, with syntax highlighting, auto-completion, and job lifecycle management.

| Setting | Value |
|---------|-------|
| Default Port | `8889` (Web UI) |
| Suggested Subdomain | `dinky` |
| Public Access | No (development tool) |
| Website | [dinky.org.cn](https://www.dinky.org.cn/) |
| Source | [GitHub](https://github.com/DataLinkDC/dinky) |

### Architecture

| Container | Image | Purpose |
|-----------|-------|---------|
| `dinky` | `nexus-dinky:1.2.5-flink1.20` | SQL IDE + built-in H2 database |

> **Custom image:** Based on `dinkydocker/dinky-standalone-server:1.2.5-flink1.20` with the Kafka SQL connector and MySQL JDBC driver pre-installed. Includes a built-in H2 database for metadata storage — no external MySQL or PostgreSQL required.

### Language

Dinky is a Chinese open-source project and defaults to Chinese (`zh-CN`). There is no environment variable or configuration file to change the default language — it is hardcoded in the compiled frontend bundle (`dinky-web/config/config.ts` → `locale.default: 'zh-CN'` with `baseNavigator: false`).

The Dinky container uses a custom entrypoint that patches `index.html` at startup to inject a script setting `umi_locale=en-US` in the browser's `localStorage`. The Umi framework (which powers Dinky's frontend) reads `localStorage` with the highest priority, so the UI loads in English on the first visit. If a user manually switches the language, their choice is preserved.

### Configuration

After first login, connect Dinky to your Flink cluster:

1. Go to **Settings** → **Cluster Management** → **Add**
2. Set the REST address: `http://flink-jobmanager:8081`
3. Save and verify the connection

### Credentials

- **Username:** `admin`
- **Password:** Generated by OpenTofu, available in Infisical under `dinky/DINKY_PASSWORD`

### Resource Limits

| Container | CPU Limit | Memory Limit | CPU Reserved | Memory Reserved |
|-----------|-----------|-------------|-------------|-----------------|
| `dinky` | 2 | 2.5 GB | 0.25 | 512 MB |

### Usage

1. Enable Dinky and Flink in the Control Plane
2. Access the SQL editor at `https://dinky.YOUR_DOMAIN`
3. Login with credentials from Infisical
4. Add the Flink cluster (see Configuration above)
5. Create a new Flink SQL task and start writing queries

### Example: First Flink SQL Query

```sql
-- Create a table with generated data
CREATE TABLE datagen_source (
    id INT,
    name STRING,
    ts TIMESTAMP(3)
) WITH (
    'connector' = 'datagen',
    'rows-per-second' = '5'
);

-- Query the stream
SELECT * FROM datagen_source;
```
